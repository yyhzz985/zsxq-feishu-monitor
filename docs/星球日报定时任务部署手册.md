# 星球日报定时任务部署手册

目标：服务器每天北京时间早上 5 点自动执行一次，抓取前一天「击球区小能手的星球」全部帖子和完整评论，生成 Markdown 和 PDF，把两个文件发给你的飞书账号，并把 PDF 插入指定飞书文档。

## 1. 文件

仓库新增：

- `src/zsxq_daily_report.py`：日报主脚本
- `deploy/zsxq-daily-report.env.example`：配置模板
- `deploy/zsxq-daily-report.service`：systemd 任务
- `deploy/zsxq-daily-report.timer`：每天北京时间 5 点触发
- `deploy/setup_zsxq_daily_report.sh`：安装脚本

这个任务和原来的 `zsxq-poll.timer` 是两个独立服务，不会停止原来的实时监控任务。

## 2. 安装

在服务器仓库目录执行：

```bash
chmod +x deploy/setup_zsxq_daily_report.sh
sudo ./deploy/setup_zsxq_daily_report.sh
```

脚本会安装到：

```text
/opt/zsxq-daily-report/app/zsxq_daily_report.py
/opt/zsxq-daily-report/config/.env
/opt/zsxq-daily-report/venv/
/data/zsxq-daily-report/
```

服务器需要 Python 3.9+。Alibaba Cloud Linux 3 可先装：

```bash
sudo dnf install -y python3.11 python3.11-pip
```

## 3. 配置

编辑：

```bash
sudo nano /opt/zsxq-daily-report/config/.env
```

必须填写：

```bash
ZSXQ_ACCESS_TOKEN=你的知识星球cookie token
ZSXQ_DAILY_FEISHU_DOC_URL=要插入PDF的飞书docx链接
FEISHU_APP_ID=飞书应用App ID
FEISHU_APP_SECRET=飞书应用App Secret
ZSXQ_DAILY_FEISHU_RECEIVE_ID_TYPE=union_id
ZSXQ_DAILY_FEISHU_RECEIVE_ID=你的飞书union_id
```

固定配置已经写好：

```bash
ZSXQ_DAILY_GROUP_ID=51115584422414
ZSXQ_DAILY_GROUP_NAME=击球区小能手的星球
ZSXQ_DAILY_TIMEZONE=Asia/Shanghai
```

个人账号优先用 `union_id`。`open_id` 和具体飞书应用绑定，不同应用之间不能混用。

如果你想发到一个飞书群，而不是个人账号，可以改成：

```bash
ZSXQ_DAILY_FEISHU_RECEIVE_ID_TYPE=chat_id
ZSXQ_DAILY_FEISHU_RECEIVE_ID=oc_xxx
```

## 4. 飞书文档上传授权

PDF 插入飞书文档使用 `lark-cli docs +media-insert`。

服务器默认用 bot 身份，不需要每天扫码。先用 `.env` 里的飞书应用初始化 `lark-cli`：

```bash
lark-cli config init --app-id cli_xxx --app-secret-stdin --brand feishu
```

然后确认飞书文档已经给这个 bot 或应用编辑权限：

```bash
ZSXQ_DAILY_LARK_AS=bot
```

如果 systemd 找不到 `lark-cli`，把 `.env` 里的 `ZSXQ_DAILY_LARK_CLI` 改成绝对路径，例如：

```bash
ZSXQ_DAILY_LARK_CLI=/www/server/nodejs/v20.20.2/bin/lark-cli
```

如果改用用户身份，服务器需要安装并授权 `lark-cli`：

```bash
lark-cli auth login
```

然后 `.env` 改成：

```bash
ZSXQ_DAILY_LARK_AS=user
```

## 5. 手动测试

先跑一次，不要直接开定时：

```bash
sudo systemctl start zsxq-daily-report.service
sudo journalctl -u zsxq-daily-report.service -n 200 --no-pager
```

看到类似内容说明成功：

```text
rendered 2026-06-05 topics=8 comments=125 ...
uploaded_to: https://...
```

输出文件在：

```text
/data/zsxq-daily-report/击球区小能手的星球/
```

## 6. 启用每天北京时间 5 点

```bash
sudo systemctl enable --now zsxq-daily-report.timer
sudo systemctl list-timers --all | grep zsxq-daily-report
```

查看日志：

```bash
sudo journalctl -u zsxq-daily-report.service -n 200 --no-pager
```

停止定时：

```bash
sudo systemctl disable --now zsxq-daily-report.timer
```

## 7. 运行逻辑

每天北京时间 5 点执行：

```bash
/opt/zsxq-daily-report/venv/bin/python /opt/zsxq-daily-report/app/zsxq_daily_report.py --date yesterday --send-to-feishu
```

脚本会：

1. 按 `Asia/Shanghai` 计算昨天日期；
2. 抓取昨天所有帖子；
3. 拉取每个帖子的完整评论；
4. 校验评论总数必须和星球接口标称 `comments_count` 一致；
5. 生成 Markdown；
6. 生成 PDF；
7. 把 Markdown 和 PDF 发给你的飞书账号；
8. 把 PDF 插入指定飞书文档。

如果评论数对不上、图片加载失败、飞书发送失败，脚本会失败退出，不会假装成功。
