## 2. 飞书 OpenAPI 模式

`.env` 需要有这些配置：

```text
ZSXQ_ACCESS_TOKEN=不要写进文档
FEISHU_SEND_MODE=openapi
FEISHU_APP_ID=不要写进文档
FEISHU_APP_SECRET=不要写进文档
FEISHU_CHAT_ID=oc_xxx
```

OpenAPI 已验证：

```text
文本发送：成功
图片发送：成功
文件发送：成功
```

如果需要临时切回旧 lark-cli：

```text
FEISHU_SEND_MODE=cli
```

切换后必须运行：

```powershell
python C:\Users\1\.hermes\scripts\zsxq_monitor.py --check
```

## 4. 阿里云执行步骤

把 `E:\AI_note\Hermes\` 下的部署文件上传到服务器后执行：

```bash
chmod +x setup_server.sh
sudo ZSXQ_SERVICE_USER=$USER ./setup_server.sh
```

编辑配置：

```bash
sudo nano /opt/zsxq-monitor/config/config.json
sudo nano /opt/zsxq-monitor/config/.env
```

服务器默认使用飞书 OpenAPI，不需要 Node.js、npm、lark-cli。`.env` 里的占位符必须全部替换成真实值，否则 `--check` 会失败。

先检查，不要直接启动：

```bash
/opt/zsxq-monitor/venv/bin/python /opt/zsxq-monitor/app/zsxq_monitor.py --check
```

检查通过后启用定时器：

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now zsxq-poll.timer
sudo systemctl status zsxq-poll.timer
```

查看运行日志：

```bash
journalctl -u zsxq-poll.service -n 100 --no-pager
tail -n 100 /var/log/zsxq-monitor/zsxq_poll.log
```
