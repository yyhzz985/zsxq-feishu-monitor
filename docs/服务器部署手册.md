# ZSXQ 飞书监控 — 阿里云服务器部署手册

> 最后更新：2026-05-29  
> 服务器：116.62.173.189（阿里云轻量，Alibaba Cloud Linux 3）  
> 部署方式：paramiko SSH + SFTP 自动化

---

## 一、服务器架构

```
systemd zsxq-poll.timer (每分钟)
    │
    └─ systemd zsxq-poll.service (oneshot)
         │
         └─ /opt/zsxq-monitor/venv/bin/python
              │
              └─ /opt/zsxq-monitor/app/zsxq_monitor.py
                   │
                   ├─ ZSXQ API (Cookie 认证)
                   ├─ SQLite → /opt/zsxq-monitor/data/zsxq_monitor.db
                   ├─ 飞书 OpenAPI (自建应用)
                   ├─ 日志 → /var/log/zsxq-monitor/zsxq_poll.log
                   └─ 存档 → /data/zsxq-archive/击球区小能手/
```

---

## 二、目录结构

| 路径 | 用途 |
|------|------|
| `/opt/zsxq-monitor/app/zsxq_monitor.py` | 主脚本（SQLite 状态机 + OpenAPI） |
| `/opt/zsxq-monitor/config/config.json` | 配置文件（group_id、last_seen_time） |
| `/opt/zsxq-monitor/config/.env` | 敏感凭据（token、app id/secret、chat_id） |
| `/opt/zsxq-monitor/data/zsxq_monitor.db` | SQLite 状态数据库 |
| `/opt/zsxq-monitor/data/zsxq_monitor.lock` | 进程锁文件 |
| `/opt/zsxq-monitor/temp/` | 便签图临时目录 |
| `/opt/zsxq-monitor/venv/` | Python 虚拟环境（Pillow） |
| `/var/log/zsxq-monitor/zsxq_poll.log` | 运行日志 |
| `/data/zsxq-archive/击球区小能手/` | 图片存档（按日期分目录） |
| `/etc/systemd/system/zsxq-poll.service` | systemd oneshot service |
| `/etc/systemd/system/zsxq-poll.timer` | systemd timer（每分钟） |

## 三、日常运维命令

### 查看服务状态

```bash
# 查看定时器状态
systemctl status zsxq-poll.timer

# 查看最近一次执行
systemctl status zsxq-poll.service

# 查看数据库
/opt/zsxq-monitor/venv/bin/python /opt/zsxq-monitor/app/zsxq_monitor.py --status

# 环境自检
/opt/zsxq-monitor/venv/bin/python /opt/zsxq-monitor/app/zsxq_monitor.py --check
```

### 启停服务

```bash
# 停止
sudo systemctl stop zsxq-poll.timer

# 启动
sudo systemctl start zsxq-poll.timer

# 完全禁用（开机不启动）
sudo systemctl disable zsxq-poll.timer

# 重新启用
sudo systemctl enable --now zsxq-poll.timer
```

### 查看日志

```bash
# 应用日志
tail -f /var/log/zsxq-monitor/zsxq_poll.log

# systemd 日志
journalctl -u zsxq-poll.service -f

# 最近 20 条
journalctl -u zsxq-poll.service -n 20 --no-pager
```

### 试运行（不写库不推送）

```bash
/opt/zsxq-monitor/venv/bin/python /opt/zsxq-monitor/app/zsxq_monitor.py --dry-run
```

### 手动触发一次

```bash
/opt/zsxq-monitor/venv/bin/python /opt/zsxq-monitor/app/zsxq_monitor.py
```

## 四、从零重新部署

如果服务器重装或需要迁移，按以下步骤操作。

### 前提条件

- 服务器运行 Alibaba Cloud Linux 3 或 CentOS 8+
- 有 root 权限
- 本地项目目录 `E:\AI_note\Hermes\` 已包含最新文件

### 步骤

**1. 上传文件到服务器**

把以下文件从 `E:\AI_note\Hermes\` 上传到服务器 `/root/`：

```
zsxq_monitor.py       → 主脚本
zsxq-poll.service     → systemd service
zsxq-poll.timer       → systemd timer
```

**2. SSH 到服务器，运行初始化**

```bash
ssh root@116.62.173.189

# 创建目录
mkdir -p /opt/zsxq-monitor/{app,config,data,temp}
mkdir -p /data/zsxq-archive/击球区小能手
mkdir -p /var/log/zsxq-monitor

# 安装依赖
yum install -y wqy-microhei-fonts python3-pip
python3 -m venv /opt/zsxq-monitor/venv
/opt/zsxq-monitor/venv/bin/pip install --upgrade pip Pillow

# 复制脚本
cp /root/zsxq_monitor.py /opt/zsxq-monitor/app/

# 复制 systemd 文件
cp /root/zsxq-poll.service /etc/systemd/system/
cp /root/zsxq-poll.timer /etc/systemd/system/
```

**3. 创建配置**

```bash
# 配置文件
cat > /opt/zsxq-monitor/config/config.json << 'EOF'
{
  "group_id": 51115584422414,
  "group_name": "击球区小能手的星球",
  "fetch_count": 1,
  "last_seen_time": null
}
EOF

# 凭据文件（替换为真实值）
cat > /opt/zsxq-monitor/config/.env << 'EOF'
ZSXQ_ACCESS_TOKEN=<你的token>
FEISHU_CHAT_ID=oc_7759cabf77c5502c6a1910aff10b3229
# 可选：额外正式内容群，多个用英文逗号分隔
# FEISHU_CONTENT_CHAT_IDS=oc_formal_xxx
# 可选：测试/告警群；告警和健康日报只发这里
# FEISHU_ALERT_CHAT_ID=oc_test_xxx
WATERMARK_TEXT=更新加V：237219265
ZSXQ_SAVE_DIR=/data/zsxq-archive/击球区小能手
FEISHU_SEND_MODE=openapi
FEISHU_APP_ID=<你的飞书app_id>
FEISHU_APP_SECRET=<你的飞书app_secret>
EOF

chmod 600 /opt/zsxq-monitor/config/.env
```

发送规则：星球图片和附件会发到 `FEISHU_CHAT_ID`、`FEISHU_CONTENT_CHAT_IDS`、`FEISHU_ALERT_CHAT_ID`；告警和健康日报只发到 `FEISHU_ALERT_CHAT_ID`。正式群只应放在内容群位置，不能放在告警群位置。

**4. 验证并启动**

```bash
# 自检
/opt/zsxq-monitor/venv/bin/python /opt/zsxq-monitor/app/zsxq_monitor.py --check

# 试运行
/opt/zsxq-monitor/venv/bin/python /opt/zsxq-monitor/app/zsxq_monitor.py --dry-run

# 一切正常后启用
systemctl daemon-reload
systemctl enable --now zsxq-poll.timer
systemctl status zsxq-poll.timer
```

## 五、关键修复记录

以下 Bug 已在当前版本修复，部署时无需额外处理：

| 问题 | 修复 |
|------|------|
| ZSXQ 图片 API `original` 字段消失 | 改用 `large` 优先，回退 `original`/`thumbnail` |
| Linux 找不到 Windows 字体 | 加 `/usr/share/fonts/wqy-microhei/wqy-microhei.ttc` 回退 |
| ZSXQ 间歇 1059 导致图片拉取失败 | 重试 3→5 次，1059 单独等 2s，间隔 0.2→1.0s |
| 文件下载接口同样问题 | 同样加 5 次重试 + 1059 特殊处理 |
