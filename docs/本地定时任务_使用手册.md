## 一、系统架构

```
Windows 任务计划 (每分钟)
    │
    └─ wscript.exe
         │
         └─ zsxq_monitor_run.vbs      ← 无窗口包装器，WshShell.Run(..., 0)
              │
              └─ pythonw.exe            ← Python 无窗口解释器
                   │
                   └─ zsxq_monitor.py   ← 主脚本（SQLite 状态机 + OpenAPI 发送）
                        │
                        ├─ ZSXQ API ──→ 拉取星球新帖
                        ├─ SQLite  ──→ 记录每条帖子的状态（discovered/rendered/sent/failed）
                        ├─ 飞书 OpenAPI ──→ 发送便签图/附件到群
                        └─ 日志文件 ──→ C:\Users\1\.hermes\logs\zsxq_poll.log
```

**为什么用 VBS 包装器？**
- `python.exe` 会弹出控制台窗口，每分钟一次打断工作。
- `pythonw.exe` 在 Task Scheduler 的 `InteractiveToken` 模式下仍可能短暂闪现。
- VBS `WshShell.Run(..., 0)` + `wscript.exe` 是 Windows 上最彻底的隐藏执行方式。

**为什么用 pythonw.exe？**
- Python 官方提供的无窗口解释器，和 `python.exe` 功能完全一致，只是不创建控制台窗口。
- 脚本日志写入文件，不需要控制台输出。

## 二、日常操作

### 查看状态

```powershell
# 查看任务计划状态
Get-ScheduledTask -TaskName 'ZSXQ-Feishu-Monitor' | Select TaskName, State

# 查看最近运行结果
Get-ScheduledTaskInfo -TaskName 'ZSXQ-Feishu-Monitor' | Format-List
```

### 关闭定时任务

```powershell
Disable-ScheduledTask -TaskName 'ZSXQ-Feishu-Monitor'
```

关闭后任务计划不再触发，不会拉取星球、不会发送飞书消息。

### 开启定时任务

```powershell
Enable-ScheduledTask -TaskName 'ZSXQ-Feishu-Monitor'
```

开启后按配置频率（当前 1 分钟）恢复运行。任务会从上次 `last_seen_time` 继续，不会漏帖也不会重复推送。

### 手动触发一次（不改变定时）

```powershell
Start-ScheduledTask -TaskName 'ZSXQ-Feishu-Monitor'
```

不会影响下一次自动触发时间。


## 五、核心原理

### 状态机（SQLite）

每条帖子有状态流转：

```
discovered -> rendered -> sent     (成功)
                       -> failed   (失败，下轮重试)
```

- **幂等键**：飞书消息携带 `zsxq-topic-{topic_id}` 幂等键，避免重试时重复推送。
- **附件独立**：附件（文件）单独记录状态，附件失败不强制重发已成功的便签图。
- **游标安全**：拉取超过 50 页不推进 `last_seen_time`，防止离线过久漏帖。

### 发送模式

当前使用 **飞书 OpenAPI**（`FEISHU_SEND_MODE=openapi`），不再依赖 `lark-cli`/Node.js：
- 文本、图片、文件均通过飞书自建应用 API 发送
- 凭据存储在 `.env`（`FEISHU_APP_ID`、`FEISHU_APP_SECRET`）

### 锁文件

`zsxq_poll_击球区.lock` 防止同一脚本并发运行。锁内记录 PID，若 PID 已不存在则自动清理。

### 告警

以下情况会发送文本告警到飞书群：
- ZSXQ token 失效
- 连续拉取失败
- 飞书发送连续失败
- 磁盘空间低于 10GB
- ZSXQ 返回 `code=1059`（非官方工具访问限制）

## 六、常见问题

**Q: 1059 错误是什么？**  
ZSXQ 接口对非官方工具的限制。脚本会自动重试，同一天同一错误只告警一次。偶尔出现不影响正常推送。

**Q: 如何切换回 lark-cli 发送？**  
修改 `.env`：`FEISHU_SEND_MODE=cli`（需确保 `lark-cli` 可用）。

**Q: 日志文件太大怎么办？**  
目前日志约 15KB，正常增长很慢。如需清理，停止任务后删除即可，不影响数据库状态。
